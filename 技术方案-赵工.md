## 1. 技术选型对比

### 1.1 候选方案

#### 方案 A：Python FastAPI + SQLite + MPA 保留（推荐）

| 维度 | 评分 | 说明 |
|------|------|------|
| **开发成本** | ⭐⭐⭐⭐⭐ (5/5) | 前端复用现有 HTML/CSS/JS 原型，后端 Python 全栈团队熟悉，无需学习新语言 |
| **部署难度** | ⭐⭐⭐⭐☆ (4/5) | SQLite 零运维，FastAPI + uvicorn 单进程部署，macOS/Linux 原生支持 |

| **数据安全** | ⭐⭐⭐⭐☆ (4/5) | SQLite 文件权限控制 + JWT 认证 + HTTPS；本地部署数据不出网 |
| **总分** | **22/25** | |

| 维度 | 评分 | 说明 |

### 1.2 推荐：方案 A — Python FastAPI + SQLite + MPA 保留

**核心理由**

1. **零成本迁移原型** — 现有 4 个 HTML 页面（login/dashboard/projects/project-detail）可直接保留为 Jinja2 模板，CSS/JS 几乎不改
2. **Hermes Agent 原生对接** — 同为 Python 生态，Hermes 可直接调用系统 API（`requests` → FastAPI），最低集成成本
3. **渐进式升级** — 第一阶段用 SQLite 上线，后续按需升 PostgreSQL，业务代码几乎不变（SQLAlchemy ORM 抽象）
4. **macOS 本地优先** — 开发和生产环境一致，6 个 Agent 日常工作流无需依赖外部云服务

### 2.2 前端策略：保留 MPA，渐进增强

**为什么保留 MPA（不转框架）**

| 维度 | MPA (保留) | 转 SPA (React/Vue) |
|------|-----------|-------------------|
| 迁移成本 | 0 — 直接复用原型 | 高 — 全部重写 |
| SEO | ✅ 原生支持 | ❌ 需 SSR |
| 初次加载 | ✅ 单个页面轻量 | ❌ JS bundle 大 |
| 后续增强 | ✅ 可逐步引入 HTMX/Alpine | ❌ 一次性投入 |
| 团队技能 | ✅ 前端同学已熟悉 | ❌ 需学习框架 |

**渐进增强路线**

```
Phase 1: 保留现有 HTML + Jinja2 模板渲染
         ├── 页面：从静态 HTML 改为 Jinja2 模板
         ├── data.js：改为 fetch API 调用后端
         └── 新增：Toast/Modal 复用原型代码

Phase 2: 引入 HTMX（可选）
         ├── 看板拖拽排序
         ├── 内联编辑
         └── 实时更新

Phase 3: 按需引入前端框架（仅复杂页面）
         └── 如仪表盘统计图表用 ECharts
```

### 3.2 Hermes Agent 自动创建/更新任务

Hermes Agent 通过 `terminal()` 或 `delegate_task()` 调用系统 API，实现任务的自动创建和更新。

**方案一：Hermes Agent 直接调用 HTTP API（推荐）**

在 Hermes 中安装一个技能，让 Agent 通过 `http` 工具调用晴天看板 API。

```python
# Hermes Agent 侧 —— 通过 delegate_task 或 terminal 执行
import httpx
import json

API_BASE = "http://localhost:8000/api"
API_KEY = "sk-..."  # 从 .env 读取

def feishu_create_task(project_id, title, priority, assignee, due, desc):
    """Hermes Agent 通过 API 自动创建任务，并同步飞书"""
    resp = httpx.post(
        f"{API_BASE}/feishu/sync-task",
        headers={"X-API-Key": API_KEY},
        json={
            "project_id": project_id,
            "title": title,
            "priority": priority,
            "assignee": assignee,
            "due": due,
            "description": desc,
            "source": "hermes_agent",   # 标记来源
            "agent_name": "赵工",        # 哪个 Agent 创建的
        }
    )
    return resp.json()
```

**Hermes cron 定时同步**

**方案二：Hermes Agent 直写 SQLite（开发环境快速验证）**

### 3.3 任务状态变更通知飞书

#### 3.3.1 后端主动推送 (Webhook)

当任务状态变更时，晴天看板后端调用飞书 Webhook 发送通知：

```python
# 晴天看板后端 —— 状态变更飞书通知
import httpx

FEISHU_WEBHOOK_URL = "https://open.feishu.cn/open-apis/bot/v2/hook/xxx"

async def notify_feishu(event_type: str, task: dict, user: str):
    """任务状态变更时通知飞书群"""
    status_emoji = {"todo": "📋", "in_progress": "🔄", "done": "✅"}
    emoji = status_emoji.get(task["status"], "📝")
    
    payload = {
        "msg_type": "interactive",
        "card": {
            "header": {
                "title": {"tag": "plain_text", "content": f"{emoji} 任务状态变更"},
                "template": "blue"
            },
            "elements": [
                {"tag": "markdown", "content": f"**{task['title']}**"},
                {"tag": "markdown", "content": f"状态: {status_emoji.get(task['status'])}"},
                {"tag": "markdown", "content": f"负责人: {task['assignee']}"},
                {"tag": "markdown", "content": f"操作人: {user}"},
                {"tag": "markdown", "content": f"截止: {task.get('due', '未设置')}"},
            ]
        }
    }
    
    async with httpx.AsyncClient() as client:
        await client.post(FEISHU_WEBHOOK_URL, json=payload)
```

#### 3.3.2 Feishu Open API 双向同步

通过飞书开放平台的自定义机器人 + 事件订阅，实现双向同步：

| 方向 | 触发条件 | 动作 |
|------|---------|------|
| 晴天 → 飞书 | 任务创建/更新/状态变更 | 发送飞书消息卡片到指定群 |
| 晴天 → 飞书 | 每日 09:00 (Hermes cron) | 发送当日待办汇总到飞书 |
| 飞书 → 晴天 | 飞书消息/指令 | 飞书机器人回调 → 更新任务 |
| 飞书 → 晴天 | 飞书多维表格变更 | 通过 Feishu Webhook 事件订阅同步 |

#### 3.3.3 通知规则

| 事件 | 目标 | 格式 |
|------|------|------|
| 任务创建 | 项目群 | 消息卡片：标题+负责人+优先级+截止日期 |
| 任务状态变更 | 项目群 + @负责人 | 消息卡片：变更摘要 |
| 每日待办汇总 | 项目群 | 消息卡片：今日待办+到期提醒 |
| Hermes Agent 创建任务 | 项目群 | 标注「🤖 Agent: 赵工 创建」 |

### 4.2 主要数据表结构

#### 4.2.1 `users` — 用户表

```sql
CREATE TABLE users (
    id            TEXT PRIMARY KEY DEFAULT (lower(hex(randomblob(16)))),
    email         TEXT NOT NULL UNIQUE,
    name          TEXT NOT NULL,

    role          TEXT NOT NULL DEFAULT 'member'  -- 'admin', 'member'

    avatar        TEXT,                           -- 首字母或头像URL

CREATE INDEX idx_users_email ON users(email);

    user_id     TEXT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    role        TEXT NOT NULL DEFAULT 'member'
                    CHECK(role IN ('owner', 'member')),
    joined_at   TEXT NOT NULL DEFAULT (datetime('now')),
    PRIMARY KEY (project_id, user_id)
);
```

#### 4.2.4 `tasks` — 任务表

```sql
CREATE TABLE tasks (
    id            TEXT PRIMARY KEY DEFAULT (lower(hex(randomblob(16)))),
    pid           TEXT NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
    title         TEXT NOT NULL,
    status        TEXT NOT NULL DEFAULT 'todo'
                      CHECK(status IN ('todo', 'in_progress', 'done')),
    priority      TEXT NOT NULL DEFAULT 'medium'
                      CHECK(priority IN ('high', 'medium', 'low')),
    assignee_id   TEXT REFERENCES users(id) ON DELETE SET NULL,
    due_date      TEXT,                                  -- 'MM-DD' 或 ISO date
    description   TEXT DEFAULT '',
    feishu_msg_id TEXT,                                  -- 飞书消息ID，用于更新/撤回
    source        TEXT DEFAULT 'manual'                   -- 'manual', 'hermes_agent', 'feishu_sync'
                      CHECK(source IN ('manual', 'hermes_agent', 'feishu_sync')),
    created_at    TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at    TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE INDEX idx_tasks_project     ON tasks(pid);
CREATE INDEX idx_tasks_assignee    ON tasks(assignee_id);
CREATE INDEX idx_tasks_status      ON tasks(status);
CREATE INDEX idx_tasks_project_status ON tasks(pid, status);
```

#### 4.2.5 `comments` — 评论表

```sql
CREATE TABLE comments (
    id          TEXT PRIMARY KEY DEFAULT (lower(hex(randomblob(16)))),
    task_id     TEXT NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
    author_id   TEXT NOT NULL REFERENCES users(id),
    text        TEXT NOT NULL,
    created_at  TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE INDEX idx_comments_task ON comments(task_id);
```

#### 4.2.6 `feishu_sync_log` — 飞书同步日志表

```sql
CREATE TABLE feishu_sync_log (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    event_type  TEXT NOT NULL,        -- 'task_created', 'task_updated', 'status_changed'
    task_id     TEXT REFERENCES tasks(id) ON DELETE SET NULL,
    feishu_msg_id TEXT,
    direction   TEXT NOT NULL         -- 'to_feishu', 'from_feishu'
                    CHECK(direction IN ('to_feishu', 'from_feishu')),
    status      TEXT NOT NULL DEFAULT 'success'
                    CHECK(status IN ('success', 'failed')),
    error_msg   TEXT,
    created_at  TEXT NOT NULL DEFAULT (datetime('now'))
);
```

### 4.3 字段映射（原型 → 生产）

| 原型字段 (data.js) | 生产字段 | 变化 |
|--------------------|---------|------|
| `TASKS[].id` | `tasks.id` | 保持 t1/t2 格式 → 改为 UUID |
| `TASKS[].pid` | `tasks.pid` | ✅ 保留 |
| `TASKS[].assignee` | `tasks.assignee_id` | 改为外键关联 users.id |
| `TASKS[].due` | `tasks.due_date` | 命名规范化 |
| `PROJECTS[].tasks` | 删除 | 改为实时 COUNT 计算 |
| `PROJECTS[].members` | `project_members` 表 | 独立关联表 |
| `COMMENTS[].author` | `comments.author_id` | 改为外键关联 |
