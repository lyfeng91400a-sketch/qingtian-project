### 1.3 核心功能

#### A. 人员概览行（卡片式汇总）

每行展示一个 Agent 的汇总信息：

| 字段 | 类型 | 说明 |
|------|------|------|
| 头像 | 圆形头像 | 同现有系统样式，background gradient |
| 姓名 | text | Agent 名称 |
| 进行中 | 数字 + 徽章 | 当前 `in_progress` 的任务数 |
| 待办 | 数字 + 徽章 | 当前 `todo` 的任务数 |
| 已完成 | 数字 + 徽章 | 当前 `done` 的任务数（今日/本周） |
| 负载指数 | 进度条 | 基于进行中+待办的任务数，0-5 级可视化 |
| 状态标签 | tag | 「过载 / 正常 / 空闲」自动判定 |

**负载指数计算规则：**

```
负载指数 = min(5, 进行中任务数 × 2 + 待办任务数 × 1)
状态判定:
  - 过载（>=4）：红色高亮，显示警告图标
  - 正常（2-3）：黄色提示
  - 空闲（0-1）：绿色，浅色显示
```

#### B. 筛选与排序

| 筛选维度 | 说明 |
|----------|------|
| 状态筛选 | 全部 / 待办 / 进行中 / 已完成 |
| 负责人筛选 | 按 Agent 姓名过滤（或全选） |
| 项目筛选 | 按项目过滤（穗穗念 / 小红书运营 / ...） |
| 排序方式 | 按截止日期升序 / 按优先级 / 按更新时间 |
| 搜索 | 任务标题关键词搜索 |

#### C. 交互操作

| 操作 | 说明 |
|------|------|
| 点击人员行 | 展开/收起该成员的任务列表 |
| 点击任务 | 跳转到项目详情页 `project-detail.html?id={pid}&task={taskId}` |
| 状态切换 | 任务列表内直接点击状态图标循环切换 todo ↔ in_progress ↔ done |
| 优先级着色 | 高优先级任务红色标签，行背景微红 |

### 1.4 "谁空闲谁过载" 判定逻辑

复用上述负载指数，另外增加维度：

| 维度 | 判定依据 | 展示形式 |
|------|---------|---------|
| 今日截止任务数 | due_date = 今日 的任务数 | 红色警告角标 |
| 超过截止任务数 | due_date < 今日 且 status != done | 🔴 逾期标签 |
| 近 3 天无活跃 | updated_at > 3 天前 无变更记录 | ⏸ 低活跃标签 |
| 负载均衡提醒 | 某 Agent 负载指数 >= 4 且另一 Agent < 1 | 横向对比气泡提示 |

### 2.3 核心功能

#### A. 工作负载视图（默认）

**可视化设计：**
- 水平进度条表示每个 Agent 的负载（0-100%）
  - 0-30% = 绿色（空闲，可承接新任务）
  - 31-60% = 黄色（正常）
  - 61-80% = 橙色（偏满）
  - 81-100% = 红色（过载）
- 显示 "当前任务数 / 建议上限"（建议上限默认为 5）
- 点击任意 Agent 行展开其任务列表

**负载均衡判定：**
- 系统自动计算每个 Agent 的负载百分比
- 当负载 > 上限 80% 时，在界面顶部显示提示条：「⚠️ 童七七当前负载较高（4/5），建议分配任务给空闲成员（二哥仅1个任务）」

#### B. 时间线视图（甘特图风格）

**可视化设计：**
- 横向时间轴（按日/周刻度）
- 每个任务显示为时间条，横跨起止日期
- 时间条颜色：按照任务优先级（高=红，中=黄，低=绿）
- 每个 Agent 一行，按姓名分组
- 拖动任务条两端调整起止日期
- 拖拽任务条到另一 Agent 行实现重新分配

#### C. 拖拽调整任务分配（v1.0 核心交互）

| 操作 | 说明 | 触发 API |
|------|------|---------|
| 拖拽任务到另一 Agent 行 | 从当前 Agent 的任务列表拖到另一 Agent 的负载条 | `PUT /api/tasks/{id}` (assignee_id) |
| 点击修改负责人 | 点击任务的负责人字段，弹出下拉选择 | - |
| 批量重新分配 | 选中多个任务 → 选择目标负责人 | 后续版本 |

**拖拽交互细节：**
1. 用户拖拽任务卡片到目标 Agent 区域
2. 目标区域高亮，显示 "+1" 占位
3. 松开后弹出确认对话框：「将「任务标题」分配给 蒋小花？」
4. 确认后调用 API 更新任务 assignee_id
5. 两个 Agent 的负载条实时刷新
6. 飞书发送通知：「童七七 将「7月内容日历排期」重新分配给 蒋小花」

### 2.4 交互细节

| 场景 | 交互反馈 |
|------|---------|
| 拖拽中 | 卡片半透明跟随鼠标，目标区域高亮 |
| 拖拽悬停 | 显示 "释放即可分配" 提示 |
| 分配成功 | Toast 通知 + 负载条实时动画重绘 |
| 分配失败 | Toast 红色错误提示 |
| 点击任务 | 弹窗展示任务详情，含编辑入口 |

## 后端实现要点

### 现有 model 无需修改

现有 `User`、`Task`、`Project`、`ProjectMember` 模型已满足团队管理需求，只需新增路由。

### 后端新增文件

```
backend/routes/team.py  ← 新增：团队管理相关路由
```

在 `main.py` 中注册：
```python
from routes.team import router as team_router
app.include_router(team_router)
```

### 关键查询逻辑（`team/board`）

```python
# 核心思路：循环所有活跃用户，统计每个用户的任务分布
users = db.query(User).filter(User.is_active == 1).all()
result = []
for user in users:
```

```
data-* 属性标记：

  - data-user-id: 目标 Agent ID（放置目标）

  1. dragstart: 记录 task-id 到 dataTransfer
  2. dragover: 阻止默认，目标元素添加高亮 class
  3. drop: 获取 task-id + target-user-id → 调用 reassignTask()
```
