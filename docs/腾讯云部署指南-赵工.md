## 1. 服务器选型推荐

### 1.1 最低配置推荐：轻量应用服务器

晴天项目是 **FastAPI + SQLite 单机架构**，对服务器性能要求不高，推荐使用腾讯云 **轻量应用服务器**（Lighthouse），性价比最高。

| 配置项 | 最低推荐 | 推荐 | 备注 |
|--------|---------|------|------|
| **CPU** | 1 核 | 2 核 | 1 核足够支撑 3-5 人日常使用 |
| **内存** | 1 GB | 2 GB | 1GB 起 Python + uvicorn + SQLite 约占用 200-300MB |
| **系统盘** | 40 GB SSD | 60 GB SSD | SQLite 数据库 + 日志，初期 < 1GB |
| **带宽** | 3 Mbps | 5 Mbps | MPA 页面轻量，3Mbps 足够 |
| **流量** | 500 GB/月 | 1000 GB/月 | 团队内部使用绰绰有余 |
| **操作系统** | Ubuntu 22.04 LTS | Ubuntu 22.04 LTS | 最稳定，Python 3.10 内置 |

### 1.2 可选配置方案

| 方案 | 规格 | 月费（参考） | 适用场景 |
|------|------|-------------|---------|
| **入门型** | 1核1G/40GB/3Mbps | **≈ 45 元/月** | 个人/3 人内小团队 |
| **经济型** | 2核2G/60GB/5Mbps | **≈ 80 元/月** | 推荐 — 6 Agent 团队日常使用 |
| **进阶型** | 2核4G/80GB/8Mbps | **≈ 120 元/月** | 并发量较高或大量飞书同步 |

> 💡 **建议首购：** 腾讯云轻量应用服务器 2核2G 配置，新用户首年通常有 3-5 折优惠，实际约 **40-60 元/月**。

### 1.3 购买指引

1. 打开 [腾讯云轻量应用服务器](https://console.cloud.tencent.com/lighthouse)
2. 点击「新建」
3. 地域选择：**广州** 或离你最近的区域（延迟越低越好）
4. 镜像选择：**Ubuntu 22.04 LTS**
5. 套餐选择：**2核2GB / 60GB SSD / 5Mbps**
6. 设置 root 密码（记好！后续 SSH 连接用）
7. 确认购买

## 2. 后端部署步骤

### 2.1 准备工作：SSH 连接服务器

打开终端，SSH 登录服务器（把 `<公网IP>` 和 `<密码>` 替换成你服务器实际的）：

```bash
ssh root@<公网IP>
```

首次登录会提示确认指纹，输入 `yes` 后回车，然后输入密码。

### 2.2 安装 Python 依赖

登入服务器后，一次性安装项目所需的环境：

```bash
# 更新系统包
apt update && apt upgrade -y

# 安装 Python3、pip、venv、git、nginx
apt install -y python3 python3-pip python3-venv git nginx

# 验证版本
python3 --version        # 应 >= 3.10
pip3 --version
```

### 2.3 上传项目代码

**方式一：通过 Git（推荐）**

如果你的代码在 GitHub/GitLab，直接在服务器上 clone：

```bash
cd /opt
git clone <你的仓库地址> qingtian
cd qingtian
```

**方式二：通过 scp 上传（本地代码直接传送）**

在你 **本地电脑** 的终端执行（不是在服务器上）：

```bash
# 将整个项目打包
cd /Users/fengliuyi/Hermes/晴天项目
tar -czf qingtian.tar.gz --exclude='.git' --exclude='__pycache__' --exclude='venv' .

# 上传到服务器（把 <公网IP> 替换成你的服务器 IP）
scp qingtian.tar.gz root@<公网IP>:/opt/

# 然后 SSH 到服务器解压
ssh root@<公网IP>
cd /opt
tar -xzf qingtian.tar.gz -C qingtian 2>/dev/null || {
  mkdir -p qingtian
  tar -xzf qingtian.tar.gz -C qingtian
}
rm -f qingtian.tar.gz
```

### 2.4 安装 Python 虚拟环境与依赖

```bash
cd /opt/qingtian/backend

# 创建 Python 虚拟环境
python3 -m venv venv

# 激活虚拟环境
source venv/bin/activate

# 更新 pip
pip install --upgrade pip

# 安装项目依赖
pip install -r requirements.txt

# 验证安装
pip list
```

### 2.5 设置环境变量（可选但推荐）

```bash
cd /opt/qingtian/backend

# 创建 .env 文件（也可以直接设环境变量）
echo 'QINGTIAN_SECRET_KEY=你的随机密钥' >> .env
```

生成一个随机密钥：

```bash
python3 -c "import secrets; print(secrets.token_hex(32))"
```

把输出的值替换上面命令中的「你的随机密钥」。

### 2.6 首次启动测试

```bash
cd /opt/qingtian/backend
source venv/bin/activate

# 前台启动测试（按 Ctrl+C 停止）
python3 -m uvicorn main:app --host 0.0.0.0 --port 8000
```

看到类似以下输出说明启动成功：

```
INFO:     Started server process [12345]
INFO:     Waiting for application startup.
INFO:     晴天看板后端启动完成
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000
```

> ⚠️ **注意：** SQLite 数据库会自动创建在 `backend/qingtian.db`，首次启动会自动创建表。

**健康检查验证：**

在另一个终端窗口 SSH 到服务器，运行：

```bash
curl http://localhost:8000/api/health
```

应返回：

```json
{"status": "ok", "version": "1.0.0"}
```

### 2.7 生成初始管理员用户

项目使用 **FastAPI 注册接口**，启动后用浏览器访问：

```
http://<公网IP>:8000/docs
```

在 Swagger UI 中找到 `POST /api/auth/register`，填入邮箱和密码即可注册第一个管理员。

或者用 curl 创建：

```bash
curl -X POST http://localhost:8000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@qingtian.com","name":"管理员","password":"admin123"}'
```

## 3. 进程守护

推荐使用 **systemd**（Ubuntu 自带）来做进程守护，服务器重启后自动拉起服务。

### 3.1 创建 systemd 服务文件

```bash
sudo vim /etc/systemd/system/qingtian.service
```

粘贴以下内容（替换实际路径）：

```ini
[Unit]
Description=晴天项目看板 - FastAPI Backend
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/qingtian/backend
ExecStart=/opt/qingtian/backend/venv/bin/uvicorn main:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=5
Environment=QINGTIAN_SECRET_KEY=你的随机密钥
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

> 把 `QINGTIAN_SECRET_KEY` 替换成前面生成的随机密钥。如果不设，项目会使用代码中的开发环境默认密钥，**生产环境必须改！**

### 3.2 启动并启用服务

```bash
# 重新加载 systemd
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start qingtian

# 设置开机自启
sudo systemctl enable qingtian

# 查看状态
sudo systemctl status qingtian

# 查看实时日志
sudo journalctl -u qingtian -f
```

### 3.3 常用管理命令

```bash
# 查看状态
sudo systemctl status qingtian

# 重启
sudo systemctl restart qingtian

# 停止
sudo systemctl stop qingtian

# 查看最近日志
sudo journalctl -u qingtian -n 100 --no-pager

# 实时查看日志
sudo journalctl -u qingtian -f
```

### 3.4 替代方案：Supervisor

如果你更熟悉 Supervisor，也可以使用：

```bash
# 安装 supervisor
apt install -y supervisor

# 创建配置
cat > /etc/supervisor/conf.d/qingtian.conf << 'EOF'
[program:qingtian]
command=/opt/qingtian/backend/venv/bin/uvicorn main:app --host 0.0.0.0 --port 8000
directory=/opt/qingtian/backend
user=root
autostart=true
autorestart=true
stdout_logfile=/var/log/qingtian/stdout.log
stderr_logfile=/var/log/qingtian/stderr.log
environment=QINGTIAN_SECRET_KEY="你的随机密钥"
EOF

# 创建日志目录
mkdir -p /var/log/qingtian

# 重载并启动
supervisorctl reread
supervisorctl update
supervisorctl start qingtian

# 查看状态
supervisorctl status
```

## 4. 前端配置修改

### 4.1 修改 config.js

项目使用 `config.js` 统一管理 API 地址，前端所有页面都通过该文件获取后端地址。

需要把 `config.js` 中的 `http://localhost:8000` 改为你的服务器公网 IP：

**在本地修改（推荐，改完上传）：**

修改 `/Users/fengliuyi/Hermes/晴天项目/config.js`：

```javascript
// 晴天项目 - API 配置
// const API_BASE_URL = 'http://localhost:8000';     // 开发环境
const API_BASE_URL = 'http://<公网IP>:8000';         // 生产环境（替换成实际 IP）
```

> ⚠️ **重要：** 如果配置了 Nginx 反向代理 + 域名（见下文 4.3），这里应改为域名地址：
> ```javascript
> const API_BASE_URL = 'https://qingtian.yourdomain.com';
> ```

**在服务器上直接改（应急）：**

用 `vim` 或 `sed` 在服务器上修改：

```bash
# 把 localhost:8000 替换为服务器公网 IP
sed -i 's|http://localhost:8000|http://<公网IP>:8000|g' /opt/qingtian/config.js

# 验证修改结果
cat /opt/qingtian/config.js
```

### 4.2 在服务器上用 Nginx 托管前端静态文件

前端是纯 HTML 文件，可以用 Nginx 直接托管，同时作为反向代理转发 API 请求到后端（还能省去 CORS 问题）。

**安装并配置 Nginx：**

```bash
apt install -y nginx
```

创建 Nginx 配置：

```bash
cat > /etc/nginx/sites-available/qingtian << 'EOF'
server {
    listen 80;
    server_name _;  # 如果有域名，替换为域名

    # 前端静态文件
    root /opt/qingtian;
    index index.html;

    # 前端页面路由
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理到后端
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # OpenAPI 文档
    location /docs {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
    }
    location /redoc {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
    }
    location /openapi.json {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
    }
}
EOF

# 启用站点
ln -sf /etc/nginx/sites-available/qingtian /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default

# 测试配置
nginx -t

# 重新加载 Nginx
systemctl reload nginx
```

**配置 Nginx 后，config.js 可以改回本地地址：**

因为前端文件和 API 请求现在通过 Nginx 的同一个域名/端口（80 端口），无需跨域。

```javascript
const API_BASE_URL = '';  // 空字符串 = 同源请求
```

### 4.3 配置 HTTPS（推荐，上线必备）

腾讯云轻量服务器可以免费申请 SSL 证书：

```bash
# 安装 certbot
apt install -y certbot python3-certbot-nginx

# 申请证书（需要有域名并解析到服务器 IP）
certbot --nginx -d qingtian.yourdomain.com

# 按照提示输入邮箱，同意条款，证书自动配置好
# certbot 会自动修改 Nginx 配置并启用 HTTPS
```

完成后，`config.js` 改为：

```javascript
const API_BASE_URL = 'https://qingtian.yourdomain.com';
```

> 如果暂时没有域名，也可以先走 HTTP + IP 方式使用，安全组只对团队 IP 开放。

## 5. 安全组配置

### 5.1 腾讯云控制台配置

在腾讯云轻量服务器控制台，找到你的实例，点击「防火墙」或「安全组」：

**必须开放的端口：**

| 端口 | 协议 | 来源 | 用途 |
|------|------|------|------|
| **22** | TCP | `0.0.0.0/0` 或你的本地 IP | SSH 远程连接 |
| **80** | TCP | `0.0.0.0/0` | HTTP（Nginx） |
| **443** | TCP | `0.0.0.0/0` | HTTPS（如配置了证书） |
| **8000** | TCP | `0.0.0.0/0` | **如不配 Nginx，直接暴露 API** |

### 5.2 推荐安全策略

```
# 如果直接用端口 8000（不配 Nginx）：
# ── 必需 ──
TCP:22, 8000    来源: 0.0.0.0/0

# 如果配置了 Nginx（推荐）：
# ── 必需 ──
TCP:22          来源: 0.0.0.0/0     # SSH（建议限制为你本地 IP）
TCP:80          来源: 0.0.0.0/0     # HTTP
TCP:443         来源: 0.0.0.0/0     # HTTPS（如配置）
# ── 可选（内网调试）──
TCP:8000        来源: 0.0.0.0/0     # 调试用，上线后可关闭
```

> 🔒 **安全建议：**
> - SSH 端口（22）建议只对你本地 IP 开放，避免被暴力破解
> - 端口 8000 在配置好 Nginx 后可以不在安全组开放，仅限内网访问
> - 如果不用 HTTPS，至少设置一个强密码 / 密钥登录

### 5.3 Ubuntu 防火墙（可选）

如果服务器上还开启了 ufw：

```bash
ufw allow 22/tcp
ufw allow 80/tcp
ufw allow 443/tcp
ufw enable
```

## 6. 预估成本

### 6.1 月度成本明细

| 项目 | 入门型方案 | 推荐方案 | 备注 |

### 6.2 新用户优惠参考

腾讯云轻量服务器新用户常见优惠：

| 时长 | 入门型（1核1G） | 推荐（2核2G） |
|------|----------------|--------------|

### 6.3 成本优化建议

1. **首购选年付** — 年付比月付省约 30%
2. **用完即关** — 非工作时间（周末、夜间）可以关机，关机不计费（仅收硬盘费）
3. **监控资源** — 用 `htop` 和 `df -h` 定期查看，发现资源剩余太多可以降配
4. **数据库备份** — SQLite 文件就是 `backend/qingtian.db`，定期备份到本地或 COS（对象存储，≈ 0.099 元/GB/月）

## 附录：快速部署检查清单

- [ ] SSH 能成功登录服务器
